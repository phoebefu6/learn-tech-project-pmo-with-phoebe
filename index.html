Learn Tech Project PMO with Phoebe

Run a data platform build like a pro

Six 45-minute sessions that turn a vague "get all our data into a warehouse" mandate into a controlled program. You learn the PMO tools that senior data and tech leaders actually use - charter, milestones, dependency gates, risk register, RACI, and a tracker - applied end to end to one running case: Northwind, a fictional retailer building a governed data platform from raw dump to production BI and AI/ML. By Session 6 you have built the complete PMO pack, and a method you will reuse on every big build.

📜 Session 1 · foundation

Frame the initiative & write the charter

Turn a one-line mandate into a charter: context, position, and the P1-P4 priorities (data, infra, governance, hiring). Meet the Northwind case.

▶ Start hereNo prep needed 🗺️ Session 2 · core

Milestone architecture

The milestone table that runs the whole build: ID, milestone, success criteria, key metrics, primary dependency. Ladder M1 to M6 across the DW-to-AI lifecycle.

Milestone toolkit 🔗 Session 3 · advanced

Dependencies & hard gates

Why hiring blocks every technical milestone, why governance gates the architecture, and how to find the critical path so parallel tracks do not collide.

Sequencing toolkit ⚠️ Session 4 · core

The risk register

Build R1-R8 the way boards read them: probability times impact, severity tags, and every risk wired to a mitigation, an owner, and a trigger.

Risk toolkit 👥 Session 5 · core

Ownership & RACI

Who is Responsible, Accountable, Consulted, Informed - across your team and the external source, cloud, and governance groups. Kill the "everyone owns it" trap.

Ownership toolkit 📊 Session 6 · advanced

Sizing, benchmarking & tracking

Size the scope with peer benchmarks, build the migration tracker (coverage %, quality flags), run the status cadence, and take Northwind to a production platform.

The finale
